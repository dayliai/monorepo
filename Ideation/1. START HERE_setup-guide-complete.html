Pre-Setup — Get Your Tools Ready
1
Install a package manager
Homebrew — the app store for your terminal Chocolatey or winget — install tools from the command line
What it does

A package manager downloads and installs command-line tools for you. Think of it like an app store, but for your terminal.

Why you need it

We'll use it to install the GitHub CLI, and it makes installing other dev tools much easier in the future.

⚠️ Your user account needs admin privileges to install Homebrew. If you don't have admin access, ask your IT team to grant it first.
Install Homebrew:
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
Add Homebrew to your PATH (so your terminal can find it):
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"
Verify:
$ brew --version
Homebrew 4.x.x
ℹ️ winget comes built into Windows 10/11 — you may already have it! Try running winget --version first. If it works, skip to Step 2.
If you prefer Chocolatey — open PowerShell as Administrator:
Set-ExecutionPolicy Bypass -Scope Process -Force;
[System.Net.ServicePointManager]::SecurityProtocol = ...
iex ((New-Object System.Net.WebClient).DownloadString(
'https://community.chocolatey.org/install.ps1'))
Close and reopen PowerShell, then verify:
> choco --version
2.x.x
2
Install the GitHub CLI
Interact with GitHub repos, PRs, and issues from your terminal
What it does

The GitHub CLI (gh) lets you clone repos, create pull requests, and manage issues — all without leaving your terminal.

Why we use it

It's the easiest way to authenticate with GitHub and clone private repos. One login and you're set.

brew install gh
Using winget:
winget install --id GitHub.cli
Or using Chocolatey:
choco install gh
3
Authenticate with GitHub
Link your terminal to the dayliai GitHub account
What it does

Connects your terminal to GitHub so you can clone private repos and push code without entering your password every time.

Why the browser

GitHub's browser login is the most secure method — it uses your existing session and supports two-factor authentication.

gh auth login
You'll see these prompts — choose these options:
Where do you use GitHub?GitHub.com
Preferred protocol?HTTPS
How to authenticate?Login with a web browser
ℹ️ A browser window will open. Log in with the dayliai GitHub account, not a personal account.
Verify it worked:
$ gh auth status
Logged in to github.com account dayliai
4
Clone the project
Download the Dayli AI codebase to your machine
What it does

Downloads a complete copy of the Dayli AI codebase — a pnpm monorepo with the Vite landing page, Next.js web app, API routes, database migrations, and shared packages.

Single repo

Everything lives in one monorepo — the landing page (Vite), web app (Next.js), shared design tokens, and Supabase config are all managed together with pnpm workspaces.

Navigate to where you want the project (e.g., Desktop):
cd ~/Desktop cd $HOME\Desktop
Clone the repository:
gh repo clone dayliai/monorepo
Navigate into the project:
cd monorepo
ℹ️ If the clone fails with "permission denied," go back to Step 3 and re-run gh auth login.
Setup — Configure Your Environment
5
Install Node.js v20
The JavaScript runtime that powers our build tools
What it does

Node.js runs JavaScript outside the browser — it powers the dev server, builds the app, and runs all our tooling.

Why v20 specifically

Version 20 is the current LTS (Long Term Support) release. Newer versions like v24 have compatibility issues with our build tools.

Install nvm (Node Version Manager):
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash
# Then close and reopen your terminal
Download and install nvm-windows:
winget install CoreyButler.NVMforWindows
# Then close and reopen your terminal
Install and use Node 20:
nvm install 20 && nvm use 20
Verify:
$ node -v
v20.x.x # Should start with v20
⚠️ If you skip this step, nothing else will work. The dev server, build, and package installation all require Node.
6
Install dependencies
Download all the libraries the project needs
What it does

Reads the project's package.json files and downloads all the libraries (React, Next.js, Tailwind, Supabase, etc.) into node_modules folders for each app.

Why pnpm

pnpm handles our monorepo's workspace packages — it links shared code between the landing page and web app so they stay in sync.

Run from the project root:
pnpm install --node-linker=hoisted
ℹ️ This takes 15-30 seconds. If it fails, make sure you completed Step 5 first and are using Node 20.
7
Configure environment variables
Connect the app to Supabase, Claude AI, and OpenAI
What it does

Creates a file called .env.local that stores secret credentials — the URLs and API keys that let the app talk to our database and AI services.

Why they're not in git

Credentials are sensitive. Anyone with the repo shouldn't see our keys. Each developer creates these locally using values shared through a secure channel.

⚠️ Get these values from the team lead or the shared credentials document. Do NOT share these in Slack, email, or chat.
Create apps/landing/.env (Vite — landing page):
VITE_SUPABASE_URL=paste-url-here
VITE_SUPABASE_ANON_KEY=paste-anon-key-here
Create apps/web/.env (Next.js — web app):
NEXT_PUBLIC_SUPABASE_URL=paste-url-here
NEXT_PUBLIC_SUPABASE_ANON_KEY=paste-anon-key-here
SUPABASE_SERVICE_ROLE_KEY=paste-service-key-here
ANTHROPIC_API_KEY=paste-claude-key-here
OPENAI_API_KEY=paste-openai-key-here
ℹ️ The env variable prefixes are different: the landing page uses Vite (VITE_) and the web app uses Next.js (NEXT_PUBLIC_). Both use the same Supabase URL and key values.
🔒 If you skip this step, the app will load but all database features silently fail — no solutions, no AI chat, no feedback saving. There may be no visible error.
8
Configure git identity
So your commits deploy correctly through Vercel
What it does

Sets your name and email on git commits. Vercel checks this email matches a GitHub account — if it doesn't, deployments get blocked.

Why it matters

Without this, Vercel's Hobby plan blocks deployments because it can't verify who made the commit. Your email must match your GitHub account exactly.

ℹ️ If git is not recognized, download and install it from git-scm.com/download/win first.
Set your name:
git config user.name "Your Name"
Set your email (must match your GitHub account):
git config user.email "your-github-email@example.com"
ℹ️ Use the exact email from your GitHub account: github.com/settings/emails
9
Start the dev server
Run the app locally and verify everything works
What it does

Starts local development servers — the landing page on localhost:3000 and the web app on localhost:3001. Changes you make instantly appear in the browser.

Why localhost

You develop and test locally before pushing to GitHub. When you push, Vercel automatically deploys the live version.

Landing page (Vite — dailylivinglabs.com):
pnpm dev:landing
# → http://localhost:3000
Web app (Next.js — dayliai.org):
pnpm dev:web
# → http://localhost:3001
ℹ️ Open the URL in your browser. The landing page shows "Where daily life gets hard... or easier" and the web app shows "Empower Your Daily Life."
⚠️ If you see errors about missing modules, go back to Step 6 and run pnpm install --node-linker=hoisted again.
