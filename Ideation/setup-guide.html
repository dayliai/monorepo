Dayli AI — Setup Guide

Get your machine ready to build and deploy

Already set up? Run the check script to verify:

bash scripts/setup.sh
Click to copy • Run from the project root directory
1
Install Node.js v20
The JavaScript runtime that powers our build tools
What it does

Node.js runs JavaScript outside the browser — it powers the dev server, builds the app, and runs all our tooling.

Why v20 specifically

The latest version (v24) has compatibility issues with our build tools. Version 20 is stable and tested with this project.

If you don't have nvm (Node Version Manager):
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
# Then restart your terminal
Install and use Node 20:
nvm install 20 && nvm use 20
Verify (no need to copy — just check the output):
$ node -v
v20.20.2 # Should start with v20
⚠️ If you skip this step, nothing else will work. The dev server, build, and package installation all require Node.
2
Install pnpm
Fast package manager for our monorepo
What it does

pnpm downloads and manages all the libraries our code depends on (React, Tailwind, Supabase, etc). It's like an app store for code packages.

Why not npm

npm comes with Node but pnpm is faster, uses less disk space, and handles our monorepo (multiple apps in one repo) better.

npm install -g pnpm
Verify (no need to copy — just check the output):
$ pnpm -v
10.x.x
3
Install dependencies
Download all the libraries the project needs
What it does

Reads the project's package.json files and downloads ~150 libraries into a node_modules folder. These are the building blocks our code imports.

Why --node-linker=hoisted

This flag tells pnpm to organize files in a flat structure that works with our build tools (Vite + Next.js). Without it, the apps can't find the libraries they need.

Run from the project root:
pnpm install --node-linker=hoisted
ℹ️ This takes 15-30 seconds. If it fails, make sure you completed steps 1 and 2 first.
4
Configure environment variables
Connect the app to Supabase (database + AI)
What it does

Creates two small files that store secret credentials — the URL and API key that let the app talk to our Supabase database and AI backend.

Why they're not in git

Credentials are sensitive. Anyone with the repo shouldn't see our keys. Each developer creates these locally using values shared through a secure channel.

⚠️ Get the Supabase URL and Anon Key from the team lead or your password manager. Do NOT share these in Slack, email, or chat.
Create apps/landing/.env (Vite — landing page):
VITE_SUPABASE_URL=paste-url-here
VITE_SUPABASE_ANON_KEY=paste-key-here
Create apps/web/.env (Next.js — web app):
NEXT_PUBLIC_SUPABASE_URL=paste-url-here
NEXT_PUBLIC_SUPABASE_ANON_KEY=paste-key-here
ℹ️ The env variable prefixes are different because the landing page uses Vite (VITE_) and the web app uses Next.js (NEXT_PUBLIC_). Both use the same Supabase URL and key values.
🔒 If you skip this step, the app will load but all database features silently fail — no solutions, no AI chat, no feedback saving. There may be no visible error.
5
Configure git identity
So your commits deploy correctly through Vercel
What it does

Sets your name and email on git commits. Vercel checks this email matches a GitHub account — if it doesn't, deployments get blocked.

Why it matters

Without this, Vercel's Hobby plan blocks deployments because it can't verify who made the commit. Your email must match your GitHub account exactly.

Set your name:
git config user.name "Your Name"
Set your email (must match your GitHub account):
git config user.email "your-github-email@example.com"
ℹ️ Use the exact email from your GitHub account: github.com/settings/emails
